# Data Ingestion with Gigapipe

Gigapipe provides a polyglot and robust data ingestion API - supporting logs, metrics, traces, profiling - compatibile with a wide range of protocols, agents, and APIs

<Tabs>

        🪵 **Loki**: Ready for integration with Grafana Loki-compatible sources<br/>
        🪵 **OpenTelemetry Logs**: For cross-platform, standardized logging<br/>
        🪵 **InfluxDB**: Supports log ingestion from InfluxDB sources<br/>
        🪵 **Elastic**: For structured log ingestion compatible with Elastic Stack<br/>
        🪵 **Syslog**: Standard protocol for Unix-based systems and network devices<br/>

    <Tab label="Metrics">
        💡 Gigapipe supports Metrics ingestion from multiple sources
        #### 📍 Supported Protocols
        🌡️ **Prometheus Remote Write**: For direct metric ingestion from Prometheus<br/>
        🌡️ **Graphite**: For integration with Graphite-compatible sources<br/>
        🌡️ **OpenTelemetry Metrics**: Standardized metric collection across environments<br/>
        🌡️ **InfluxDB Line Protocol**: Enables ingestion from InfluxDB-compatible sources<br/>

        #### 📍 Supported Agents
        📦 [Grafana Alloy](https://grafana.com/docs/alloy/latest/)<br/>
        📦 [Telegraf](https://www.influxdata.com/time-series-platform/telegraf/)<br/>
        📦 [Prometheus Node Exporter](https://prometheus.io/docs/guides/node-exporter/)<br/>
        📦 [Vector](https://vector.dev/)<br/>
        📦 [Amazon CloudWatch Agent](https://aws.amazon.com/cloudwatch/)<br/>

        #### 📍 [APIs](/docs/api)
        - **Prometheus PushGateway**: Used for pushing ephemeral metrics to Gigapipe<br/>
        - **HTTP/REST API**: For HTTP-based metric ingestion<br/>
        - **Graphite API**: Allows metric ingestion in Graphite format<br/>
    </Tab>

    <Tab label="Traces">
        💡 Gigapipe supports Traces ingestion from multiple sources
        #### 📍 Supported Protocols
        👣 **OpenTelemetry Traces**: Full support for cross-platform distributed tracing<br/>
        👣 **Zipkin**: Popular format using Zipkin v2 JSON<br/>

        #### 📍 Supported Agents
        📦 [OpenTelemetry Collector](https://opentelemetry.io/docs/collector/)<br/>
        📦 [Grafana Alloy](https://grafana.com/docs/alloy/latest/)<br/>
        📦 [Jaeger Agent](https://www.jaegertracing.io/)<br/>

        #### 📍 [APIs](/docs/api)
        - **Jaeger Thrift**: Supports trace ingestion in Jaeger's Thrift format<br/>
        - **HTTP/REST API**: For flexible HTTP-based trace ingestion<br/>
    </Tab>

    <Tab label="Profiling">
        💡 Gigapipe supports Continuous Profiling ingestion using Pyroscope compatibility
        #### 📍 Supported Protocols
        🔍 **Pyroscope**: Full API support for Pyroscope SDK Clients<br/>
        🔍 **OpenTelemetry Profiling**: Standardized profiling format to track application resource use<br/>

        #### 📍 Supported Agents
        📦 [Grafana Agent](https://grafana.com/docs/agent/latest/)<br/>
        📦 [Pyroscope Agent](https://pyroscope.io/)<br/>
        📦 [Parca Agent](https://www.parca.dev/)<br/>

        #### 📍 [APIs](/docs/api)
        - **HTTP/REST API**: For sending profiling data to Gigapipe<br/>
    </Tab>
</Tabs>
